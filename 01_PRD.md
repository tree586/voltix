## 1. 제품 개요

볼틱스는 전기도면을 업로드하면 도면 내 전기 심볼을 자동 인식하여 스위치, 콘센트, 통신 인출구, 통합수구, 세대분전반 등 주요 전기자재 수량을 산출하고, 산출 결과를 자재 품번·단가·견적서·발주 리스트·공급 상담까지 연결하는 전기자재 업무 자동화 플랫폼이다.

기존 전기설비 적산 업무는 도면 확인, 심볼 카운팅, 수량표 작성, 품번 대조, 단가 입력, 견적서 작성, 발주 리스트 정리까지 대부분 수작업으로 진행된다. 이 과정에서 반복 업무 시간이 많이 소요되고, 누락·중복·오산출·오입력 문제가 발생할 수 있다.

볼틱스는 단순한 도면 적산 프로그램이 아니라, 도면 분석 이후 실제 전기자재 유통·견적·공급 업무까지 연결하는 플랫폼을 지향한다.

## 2. 제품 목적

### 2.1 핵심 목적

전기도면 기반 전기자재 수량산출 업무를 자동화하고, 산출 결과를 견적·발주·공급 업무까지 연결하여 전기공사 및 전기자재 유통 업무의 효율과 정확도를 높인다.

### 2.2 세부 목적

| 구분 | 목적 |
|---|---|
| 업무 시간 절감 | 수작업 도면 카운팅 및 엑셀 정리 시간 단축 |
| 오류 감소 | 자재 수량 누락, 중복, 오산출, 오입력 최소화 |
| 견적 자동화 | 산출 수량과 단가표를 연결하여 견적서 자동 생성 |
| 공급 연결 | 확정 수량을 발주 리스트와 공급 상담으로 연결 |
| 데이터 축적 | 현장별 도면, 심볼, 자재, 단가, 견적 데이터를 누적 관리 |
| 실무 검수 지원 | 자동 인식 결과를 사용자가 검수·수정할 수 있는 구조 제공 |

## 4. 고객 문제 정의

### 4.1 현재 업무 문제

1. 전기도면을 사람이 직접 확인하며 심볼을 카운팅한다.
2. 스위치, 콘센트, 통신 인출구, 통합수구 등 반복 품목이 많아 시간이 오래 걸린다.
3. 현장마다 도면 심볼, 레이어, 블록명, 표기 방식이 달라 수량산출 기준이 일정하지 않다.
4. 도면 수량산출 이후 자재 품번, 규격, 단가, 견적서 작성 업무가 별도로 진행된다.
5. 엑셀 작업 과정에서 누락, 중복, 오입력 가능성이 있다.
6. 도면 변경 시 기존 수량을 다시 검토해야 한다.
7. 견적 담당자, 공무 담당자, 유통사 간 동일 데이터를 공유하기 어렵다.

### 4.2 핵심 Pain Point

> 전기도면에서 자재 수량을 뽑는 일은 반복적이지만, 견적·발주와 직접 연결되기 때문에 오류가 발생하면 비용과 납기 리스크로 이어진다.

## 5. 제품 포지셔닝

### 5.1 한 줄 정의

볼틱스는 전기도면에서 주요 전기자재 수량을 자동 산출하고, 자재 품번·견적·발주·공급까지 연결하는 전기자재 업무 자동화 플랫폼이다.

### 5.2 차별화 방향

| 기존 적산 프로그램 | 볼틱스 |
|---|---|
| 도면 기반 물량 산출 중심 | 수량산출 + 품번 매칭 + 견적 + 공급 연결 |
| 적산 전문가 중심 | 견적 담당자, 공무 담당자, 유통사도 사용 가능 |
| 설치형 프로그램 중심 | 웹 기반 프로젝트 관리형 플랫폼 지향 |
| 수량표 출력 중심 | 견적서, 발주 리스트, 공급 상담까지 연결 |
| 공종 전체 적산 지향 | 전기자재 특화 자동화부터 시작 |

## 6. 제품 범위

### 6.1 MVP 범위

초기 MVP는 모든 전기설비 적산을 목표로 하지 않는다. 공동주택·오피스텔 세대 전기도면에서 반복적으로 등장하는 배선기구와 통신 관련 자재 수량산출부터 검증한다.

| 구분 | MVP 범위 |
|---|---|
| 도면 형식 | DWG 우선, DXF 보조, PDF는 후속 |
| 도면 종류 | 세대 전기 평면도 |
| 적용 건물 | 공동주택, 오피스텔, 소형 주거시설 |
| 분석 단위 | 프로젝트, 도면, 세대 타입, 층, 구역 |
| 산출 품목 | 스위치, 콘센트, 통신 인출구, 통합수구, 세대분전반 |
| 출력물 | 수량표, 엑셀, 견적서, 발주 리스트 |

### 6.2 MVP 제외 범위

| 제외 항목 | 제외 사유 |
|---|---|
| 모든 공종 통합 적산 | 초기 범위 과대 |
| 건축·기계 적산 | 전기자재 특화 차별성 우선 |
| 스캔 PDF 완전 자동 인식 | 기술 난이도 높음 |
| 100% 자동 산출 보장 | 도면 표준화 차이로 검수 필요 |
| BIM 완전 연동 | 후속 고도화 범위 |
| ERP 실시간 연동 | PoC 이후 검토 |

## 8. 사용자 시나리오

### 8.1 전기공사 공무 담당자 시나리오

1. 사용자가 로그인한다.
2. 신규 프로젝트를 생성한다.
3. 현장명, 세대 타입, 도면 버전 정보를 입력한다.
4. DWG 전기도면을 업로드한다.
5. 시스템이 도면의 블록, 레이어, 심볼 정보를 분석한다.
6. 시스템이 스위치, 콘센트, 통신, 통합수구, 세대분전반 수량을 1차 산출한다.
7. 사용자는 도면 위에서 인식 결과를 검수한다.
8. 누락 또는 오인식된 항목을 수정한다.
9. 최종 수량표를 확정한다.
10. 엑셀 수량표와 견적서를 다운로드한다.
11. 필요한 경우 공급 상담을 요청한다.

### 8.2 전기자재 유통사 시나리오

1. 고객으로부터 받은 전기도면을 프로젝트에 등록한다.
2. 도면 기반으로 자재 수량을 자동 산출한다.
3. 산출 결과를 자재 품번과 매칭한다.
4. 공급 단가를 적용해 견적서를 생성한다.
5. 대체품 또는 추천 품목을 추가한다.
6. 발주 리스트를 생성하고 고객에게 전달한다.

## 13. 수용 기준

MVP가 완료되었다고 판단하기 위한 기준은 다음과 같다.

1. 사용자는 계정으로 로그인할 수 있다.
2. 사용자는 프로젝트를 생성하고 도면을 업로드할 수 있다.
3. 시스템은 DWG/DXF 도면에서 블록명, 레이어명, 좌표 정보를 추출할 수 있다.
4. 사용자는 인식된 심볼을 자재 품목과 매핑할 수 있다.
5. 시스템은 주요 전기자재 수량을 자동 산출할 수 있다.
6. 사용자는 자동 산출 결과를 검수하고 수정할 수 있다.
7. 시스템은 품목별 수량표를 생성할 수 있다.
8. 사용자는 수량표를 엑셀로 다운로드할 수 있다.
9. 사용자는 자재코드와 단가를 등록할 수 있다.
10. 시스템은 견적서와 발주 리스트를 생성할 수 있다.
11. PoC 도면 기준으로 실무자가 검수 가능한 수준의 결과를 제공한다.
