## 1. 진행 원칙

Voltix MVP는 완전 자동 적산보다 실무자가 검수 가능한 자동 수량산출 흐름을 먼저 만든다. 따라서 실행 순서는 자료 확보, 기준 정의, 기술 검증, Sprint 1 개발 착수 순서로 진행한다.

---

## 2. 전체 순서

| 순서 | 단계 | 목표 | 주요 산출물 | 상태 |
|---|---|---|---|---|

## 3. 단계별 의사결정 포인트

### 3.1 PoC 자료 확보

결정해야 할 내용:

1. 실제 DWG/DXF 샘플 도면 제공 가능 여부
2. 수작업 산출 엑셀 제공 가능 여부
3. 자재 품번표와 단가표 제공 범위
4. 도면 반출 또는 익명화 필요 여부
5. PoC 검증 담당자 지정 여부

완료 기준:

- 샘플 도면 5개 이상 확보
- 수작업 산출표 1개 이상 확보
- 주요 자재 품목표 확보
- 검증 담당자 1명 이상 지정

### 3.2 심볼·자재 기준 정의

결정해야 할 내용:

1. MVP 대상 심볼 종류
2. 심볼명, 블록명, 레이어명 키워드 기준
3. 심볼과 자재 품목의 기본 매핑 기준
4. 세대 타입별 수량 집계 기준
5. 중복 심볼 판단 기준

완료 기준:

- MVP 심볼 카테고리 확정
- 초기 매핑 규칙 초안 작성
- 미분류·검수필요 상태 기준 정의

### 3.3 CAD Parser·뷰어 기술 검토

결정해야 할 내용:

1. DWG 직접 파싱 여부
2. DWG에서 DXF로 변환 후 파싱할지 여부
3. CAD Parser 후보 라이선스 검토
4. 도면 뷰어를 SVG, Canvas, WebGL 중 무엇으로 구현할지
5. 좌표계, 확대/축소, 심볼 오버레이 방식

완료 기준:

- CAD Parser 1차 후보 선정
- 도면 뷰어 구현 방식 선정
- 샘플 도면 기준 기술 검증 계획 수립

### 3.4 Sprint 1 백로그 확정

결정해야 할 내용:

1. 인증 방식
2. 프로젝트 기본 필드
3. 도면 업로드 허용 확장자와 용량 제한
4. 파일 저장 구조
5. 도면 분석 작업 상태 모델

완료 기준:

- Sprint 1 사용자 스토리 확정
- API 초안 확정
- DB 테이블 초안 확정
- QA 시나리오 초안 확정

## 4. 권장 진행 일정

| 주차 | 목표 | 주요 작업 |
|---|---|---|
| 1주차 | 자료 확보 및 기준 정리 | 샘플 도면 요청, 자재표 요청, 심볼 기준표 초안 |
| 2주차 | 기술 검토 착수 | CAD Parser 후보 비교, 도면 뷰어 방식 검토 |

| 4주차 | Sprint 1 개발 착수 | 회원/권한, 프로젝트 생성, 도면 업로드 구현 |

## 5. 현재 우선순위

1. `06_POC_DATA_REQUEST.md`를 기준으로 샘플 자료를 요청한다.
2. 자료가 확보되기 전까지 `07_SYMBOL_MATERIAL_STANDARD_DRAFT.md` 기준표를 병행 정리한다.
3. 샘플 DWG를 확보하면 `08_TECH_REVIEW_CHECKLIST.md` 기준으로 CAD Parser 검증을 시작한다.
4. 기술 검토와 무관하게 진행 가능한 Sprint 1 공통 기능은 `09_SPRINT1_BACKLOG.md` 기준으로 착수 준비한다.
